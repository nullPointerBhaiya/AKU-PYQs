## Section 2: Short Answer Questions (Answer Any Three)

### (a) Categories of Transmission Media

**Answer:** Transmission media is the medium or path through which data is transmitted from one device to another in a network.

There are mainly **two categories** of transmission media:

#### 1. Guided Media (Wired Media)

Guided media uses physical cables to transmit data. In this type, signals travel through wires. It is more secure and less affected by noise.

**Examples of guided media:**
- Twisted pair cable
- Coaxial cable
- Optical fibre cable

*Optical fibre is very fast and uses light signals for communication.*

#### 2. Unguided Media (Wireless Media)

Unguided media does not use any physical cables. Data is transmitted through air using electromagnetic waves.

**Examples:**
- Radio waves
- Microwaves
- Satellite communication

This type is easy to install and useful for long-distance communication.

**Conclusion:** Both guided and unguided media are important for data transmission depending on requirement.

---

### (b) Network Characteristics

Network characteristics are the features that define the performance and quality of a network.

**The main characteristics are:**

1. **Speed:** How fast data is transmitted from one device to another
2. **Reliability:** The network should work properly without failure
3. **Security:** Data is protected from unauthorized access
4. **Scalability:** The network can be expanded when required
5. **Performance:** The overall working efficiency of the network

**Conclusion:** A good network should be fast, reliable, secure, and efficient.

---

### (c) What is ROM?

**💽 ROM (Read Only Memory)**

ROM is a type of memory that stores permanent data and instructions. It is non-volatile, which means it keeps data even when the power is turned off. It is mainly used to store system programs like BIOS.

**Key Features:**
- ROM is a **non-volatile memory**, which means data is not lost when the power is turned off
- Data stored in ROM is permanent and cannot be modified easily
- ROM is mainly used to store important instructions required to start the computer (booting instructions)

**Conclusion:** ROM is very important for the basic functioning and startup of a computer system.

---

### (d) Characteristics of Data Communication

Data communication is the process of transferring data from one device to another.

**The main characteristics are:**

1. **Delivery:** Data should be delivered to the correct destination
2. **Accuracy:** Data should be error-free and correct
3. **Timeliness:** Data should reach on time without delay
4. **Jitter:** Variation in delay should be minimum for smooth communication

**Conclusion:** If all these characteristics are maintained, communication becomes efficient and reliable.

---

### (e) Categories of Software

**🔹 Software**

Software is a set of programs and instructions that tells a computer what to do. It is the intangible part of a computer system, which means it cannot be seen or touched. Software helps the computer to perform different tasks.

**There are mainly two main categories:**

#### 💻 System Software
System software is a type of software that manages and controls the computer hardware. It provides basic functions and creates a platform for other software to run.

**Examples:** Operating systems (Windows, Linux), device drivers, and utility programs

#### 🧾 Application Software
Application software is designed to help users perform specific tasks. These tasks can include typing documents, watching videos, or playing games.

**Examples:** MS Word, Excel, and media players

**Conclusion:** All types of software are important for proper functioning of a computer system.

## Section 3: Generations of Computer System

### 📘 Introduction

The development of computers is divided into five generations based on the technology used. Each generation shows improvement in size, speed, storage, and reliability.

---

### 💻 1st Generation (1940–1956) – Vacuum Tubes

**👉 Technology:** Vacuum Tubes

**👉 Features:**
- Very large in size (room-sized computers)
- Produced a lot of heat
- Consumed high electricity
- Slow processing speed
- Not reliable (frequent failures)

**👉 Example:** ENIAC

---

### 💻 2nd Generation (1956–1963) – Transistors

**👉 Technology:** Transistors

**👉 Features:**
- Smaller than first generation
- Faster processing speed
- More reliable
- Less heat and power consumption
- Used assembly language

**👉 Improvement:** Better than vacuum tubes

---

### 💻 3rd Generation (1964–1971) – Integrated Circuits (IC)

**👉 Technology:** Integrated Circuits (IC)

**👉 Features:**
- Much smaller and compact
- High speed and efficiency
- More reliable
- Less heat generation
- Use of operating systems

**👉 Improvement:** Many transistors in one chip

---

### 💻 4th Generation (1971–Present) – Microprocessors

**👉 Technology:** Microprocessors

**👉 Features:**
- Very small size (PCs, laptops)
- Very fast processing
- High storage capacity
- Low cost
- Used in personal computers

**👉 Example:** Modern computers

---

### 💻 5th Generation (Present & Future) – AI & Advanced Tech

**👉 Technology:** Artificial Intelligence, Quantum Computing

**👉 Features:**
- Focus on smart computers
- Machines can think and learn
- Used in AI, robotics, automation
- Very high speed and advanced processing

**👉 Example:** AI systems, robots

## Section 4: Characteristics of a Good Programming Language

### Introduction

A programming language is used to write instructions for a computer to perform tasks. A good programming language should have some important qualities so that it is easy to use, understand, and efficient.

---

### 📘 Features of a Good Programming Language (SUPREME S)

#### 🔹 Simplicity
Simplicity means the programming language should be easy to learn and use. It should have simple syntax so that beginners can understand it quickly and write programs without confusion.

#### 🔹 Readability
Readability means the programs written in the language should be easy to read and understand. Clear structure and meaningful keywords help programmers understand the code easily.

#### 🔹 Efficiency
Efficiency means the language should use system resources properly and produce fast and optimized programs. It should not waste memory or processing power.

#### 🔹 Portability
Portability means a program written in one system can run on another system with little or no changes. This makes the language flexible and widely usable.

#### 🔹 Modularity
Modularity means the program can be divided into smaller parts or modules. These modules can be reused, making programming easier and more organized.

#### 🔹 Error Handling
Error handling means the language should be able to detect errors and handle them properly. It helps in avoiding program crashes and improves reliability.

#### 🔹 Security
Security means the language should protect data and programs from unauthorized access. It should prevent misuse and ensure safe execution of programs.

---

### Conclusion

A good programming language should be simple, efficient, reliable, and easy to maintain. These characteristics help programmers to develop better and faster programs.

## Section 5: Functions of Operating System

### Introduction

An **Operating System (OS)** is system software that acts as an interface between the user and the computer hardware. It controls and manages all the activities of a computer system. Without an operating system, a computer cannot function properly.

---

### Functions of Operating System

#### 1. Process Management
- The operating system manages all running programs (processes)
- It decides which process will run first and how much time it will get
- It also handles multitasking (running multiple programs at the same time)

#### 2. Memory Management
- The OS manages the main memory (RAM)
- It allocates memory to different programs and removes it when not needed
- This helps in efficient use of memory

#### 3. File Management
- The OS manages files and folders in the computer
- It helps in creating, deleting, reading, and writing files
- It also organizes data in a proper way

#### 4. Device Management
- The OS controls all hardware devices like keyboard, mouse, printer, and monitor
- It uses device drivers to communicate with hardware

#### 5. Security and Protection
- The OS protects data from unauthorized access
- It provides features like passwords and user authentication

#### 6. User Interface
The OS provides an interface for users to interact with the computer.

**Two types:**
- Command Line Interface (CLI)
- Graphical User Interface (GUI)

#### 7. Input and Output Management
- The OS manages input and output operations
- It controls how data is received from input devices and sent to output devices

#### 8. Error Handling
- The OS detects errors in hardware and software and takes necessary actions
- It helps in maintaining system stability

#### 9. Resource Management
- The OS manages all resources like CPU, memory, and devices
- It ensures proper and fair use of resources

---

### Conclusion

Operating system plays a very important role in the functioning of a computer. It manages hardware, software, and provides a user-friendly environment. Without OS, a computer system cannot work efficiently.

## Section 6: Input/Output Devices

### Introduction

Input and output devices are important parts of a computer system. **Input devices** are used to give data and instructions to the computer, while **output devices** are used to display the result to the user.

---

Input and output devices help in communication between user and computer. Input devices send data to the computer, and output devices display the result. Both are necessary for proper functioning of a computer system.

## Section 7: Algorithm vs Flowchart

### Introduction

Algorithm and flowchart are important tools used in programming to solve a problem. Before writing a program, we use algorithm and flowchart to understand the logic clearly. Both help in designing a solution, but they are different in form and representation.

---

### 📘 Algorithm

**🔹 Definition**

An algorithm is a step-by-step procedure used to solve a problem. It shows the sequence of steps that must be followed to get the correct result in a clear and simple way.

**Example of Algorithm (Addition of two numbers):**

```
Step 1: Start
Step 2: Input two numbers A and B
Step 3: Add A and B and store in SUM
Step 4: Display SUM
Step 5: Stop
```

---

### 📊 Flowchart

**🔹 Definition**

A flowchart is a graphical or visual representation of an algorithm. It uses different symbols to show the steps of a process in a clear and simple way.

**🔹 Common Symbols:**
- **Oval** is used to represent Start and Stop
- **Rectangle** is used for processing steps
- **Parallelogram** is used for input and output
- **Diamond** is used for decision making

**Example of Flowchart (Addition of two numbers):**

```
Start → Input A, B → A + B → Display result → Stop
```

*(It is drawn using symbols and arrows)*

---

### Conclusion

Algorithm and flowchart both are very useful in programming. Algorithm helps in writing the steps clearly, while flowchart helps in understanding the process visually. Using both together makes problem solving easy and effective.
